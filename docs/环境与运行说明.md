# 环境与运行说明

## 1. 创建环境

```powershell
conda env create -f SatPhoto-pro\environment_satphoto.yml
conda activate satphoto
```

或在已有 conda 环境中安装依赖：

```powershell
pip install -r SatPhoto-pro\photogrammetry_suite\requirements.txt
conda install -c conda-forge gdal rasterio -y   # Task2 进阶题 / Task5 需要
```

依赖清单（已验证）：numpy, scipy, opencv-python, matplotlib, tifffile, pandas, rasterio, GDAL(osgeo), PyQt5, pygeodesy, onnxruntime, python-docx。

## 2. 一个常见坑：LAPACK/GDAL 的 DLL

直接调用 conda 环境的 `python.exe` 时，若 `<env>\Library\bin` 不在 PATH，会出现：
- numpy 矩阵乘法可用，但 `np.linalg.lstsq/solve/svd` **进程崩溃**（MKL/LAPACK DLL 加载失败）；
- rasterio/GDAL 找不到 `GDAL_DATA / PROJ_LIB`。

**解决**：用本仓库提供的 `.bat` 启动（已自动把 `<env>;<env>\Library\bin;<env>\Scripts` 加入 PATH，并设置 `GDAL_DATA/PROJ_LIB`）；或先 `conda activate satphoto` 再运行。

## 3. 数据放置

把课程数据放到本仓库根目录（与 `run_pipeline.bat` 同级）：

```
全流程测试用例/        卫星摄影测量-补充数据/        实习数据/(可选)        geoid_data/(EGM2008)
```

## 4. 运行方式

| 目的 | 命令 |
|---|---|
| 图形界面（演示，含"★ 全流程"一键） | `SatPhoto-pro\启动系统.bat` |
| 测试用例全流程（命令行复现） | `run_pipeline.bat` |
| 只跑部分阶段 | `run_pipeline.bat --stages rpc dom` |
| Task4 跑全部 5 种匹配 | `run_pipeline.bat --task4-all` |
| 补充数据赛道实验 | `run_supplementary.bat` |
| 重新生成报告图件 | `make_report_figures.bat` |

## 5. 产物位置

```
SatPhoto-pro/suite_outputs/
├─ pipeline/           全流程：01_rpc_corrected / 02_staged_corrected / 03_dom / 04_tie_points / pipeline_result.json
├─ task3/ task4/ task5/  各环节中间与最终产物（核线 / 视差 / 点云 / DSM / 指标 csv）
└─ supplementary/      补充数据：BWD_DOM_srtm.tif / FWD_DOM_srtm.tif / supp_dom_compare.png / supplementary_result.json
```

## 6. 常见问题

- **GUI 启动后闪退**：查看仓库根目录 `suite_runtime.log`；多为环境 DLL 或数据路径问题。
- **Task5 报"缺少依赖"**：安装 rasterio（见第 1 节）。
- **Task4 CREStereo 不可用**：需单独放置 ONNX 权重（体积大未随仓库提供），其余四种匹配不受影响。
